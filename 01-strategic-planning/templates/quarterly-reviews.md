# Template: Quarterly PI Review

## Purpose
Formal review held at the end of every PI with skip-level 
and all managers. Makes progress visible, applies context 
to variance, and refines targets jointly.

## Format
60 minutes. All managers present. Skip-level present.
No narrative before the data. Numbers first, always.

## Pre-work (manager sends 48 hours before)
- Five primary metrics vs baseline
- List of stories tagged internal vs ecosystem
- Defect origin breakdown for the PI
- One qualitative observation per team

---

## Agenda

### 0:00 - Open (2 min)
State the purpose:
"We are here to look at what the data shows, understand 
what caused it, refine our targets, and agree what changes 
next PI. All decisions made today will be documented and 
emailed to everyone within 24 hours."

---

### 0:02 - Question 1: What did the data show? (15 min)
Present each of the five metrics against baseline.
No commentary first — just the numbers.

| Metric | Baseline | This PI | Change |
|--------|----------|---------|--------|
| Story cycle time | | | |
| PR cycle time | | | |
| Defect density | | | |
| Story reopen rate | | | |
| Test coverage | | | |

After numbers are presented, ask:
"What stands out? What surprises you?"

---

### 0:17 - Question 2: What caused variance? (15 min)
For each metric that moved significantly (up or down), ask:
"What caused this — AI adoption, PI complexity, ecosystem 
dependencies, stakeholder changes, or something else?"

Tag each cause:
- [AI] - attributable to AI intervention
- [COMPLEXITY] - PI was unusually complex or simple
- [ECOSYSTEM] - external system caused delay or defect
- [STAKEHOLDER] - direction changed mid-PI
- [OTHER] - name it

This is your attribution record. It protects the team 
when targets are later challenged.

---

### 0:32 - Question 3: What do we refine? (10 min)
For each target, state whether it should:
- Stay the same
- Move up (was too conservative, data supports it)
- Move down (external factors outside team control)
- Get a wider range (high variance, more data needed)

Rules:
- Both directions are valid
- All changes require sign-off from both sides in this room
- No refinement is made without a named reason

Record every decision.

---

### 0:42 - Question 4: What changes next PI? (10 min)
- Which interventions continue unchanged?
- Which interventions need adjustment?
- Which interventions get dropped?
- What new interventions start next PI?
- Are resources adequate or does something need to change?

---

### 0:52 - Close (8 min)
Summarise decisions made.
Confirm everyone agrees before leaving the room.
State: "I will send an email summary within 24 hours. 
If anything is wrong in that summary, tell me within 
48 hours."
